# Usage

Installation, the full CLI reference, typical workflows, the Python API, and troubleshooting.

## 1. Installation

Requirements: Python ≥ 3.10. Optional: CUDA GPU (embedding is ~10× faster), pandoc (better DOCX conversion — auto-downloaded on first use if missing).

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1        # Windows
# source .venv/bin/activate         # Linux/macOS
pip install -e .
doqqy --help
```

The first `doqqy embed` downloads ~2 GB of models (bge-m3 + reranker) from HuggingFace into the standard HF cache (`%USERPROFILE%\.cache\huggingface`, override with `HF_HOME`). Everything after that is offline.

### Environment

| Variable | Effect |
|---|---|
| `DOQQY_DEVICE` | Force `cuda` or `cpu` (default: auto-detect); applies to both embedder and reranker |
| `DOQQY_RERANKER_FP16` | Set to `1` to run the reranker in fp16 on CUDA (default: `0` — fp32 safe mode) |
| `HF_HOME` | HuggingFace model cache directory |
| `GEMINI_API_KEY` / `ANTHROPIC_API_KEY` | **Unused by the pipeline.** Reserved in `.env.example` for optional future LLM extras (`pip install -e .[llm]`); no current code path calls them |

A `.env` file at the working-directory root is loaded automatically (`python-dotenv`, no override of existing env).

## 2. Directory model — important

doqqy treats **the current working directory as the corpus root** (like git treats it as the repo). Run all commands from the folder that contains your documents:

```
my-corpus/
├── raw/            # INPUT — your original files (never modified by doqqy)
│   ├── erp12/          # subfolders become tags → --tag erp12
│   │   └── billing/    # nested → tags [erp12, billing]
│   └── notes.md
├── processed/      # OUTPUT — canonical markdown (open this as an Obsidian vault)
│   └── INDEX.md    # generated by `doqqy index`
└── .doqqy/         # STATE — chunks.parquet, store.lance/, topics.yaml, logs/
```

You can maintain any number of separate corpora — each is just a directory. Deleting `.doqqy/` + `processed/` fully resets a corpus; `raw/` is never touched.

## 3. CLI reference

### `doqqy ingest`

Convert `raw/` files (`.md`, `.markdown`, `.txt`, `.html`, `.htm`, `.pdf`, `.docx`, `.xml`, `.xlsx`, `.csv`) to canonical markdown in `processed/`.

```powershell
doqqy ingest                     # all of raw/
doqqy ingest --source ./somewhere-else
doqqy ingest --limit 5           # first 5 files only (smoke test)
```

Failures don't stop the run — a summary panel lists failed files; details in `.doqqy/logs/ingest.log`. Re-running overwrites `processed/` outputs (idempotent).

### `doqqy chunk`

Header-aware split of `processed/**/*.md` into `.doqqy/chunks/chunks.parquet`.

```powershell
doqqy chunk
doqqy chunk --processed ./other-processed-dir
```

### `doqqy embed`

Encode all chunks with bge-m3 (dense + sparse) and write the LanceDB store. **Rebuilds the whole table every time** (no incremental update yet).

```powershell
doqqy embed
$env:DOQQY_DEVICE = "cpu"; doqqy embed    # force CPU
```

### `doqqy query`

Hybrid search: dense + sparse retrieval (50 candidates each) → RRF fusion → cross-encoder rerank → top-k.

```powershell
doqqy query "how does JWT refresh work?"
doqqy query "PayTR refund flow" --top-k 10       # more results (default 5)
doqqy query "invoice states" --full              # full chunk text (default: 400-char preview)
doqqy query "invoice states" --no-rerank         # skip reranker (faster, RRF order)
doqqy query "refund process" --tag erp12         # restrict to one corpus/folder (must match ^[\w-]+\Z)
```

Each hit shows the source file, section path, and score breakdown (`dense=<rank> | sparse=<rank> | rrf=<score> | rerank=<score>`). Exit code 1 when nothing is found.

### `doqqy map`

Build the relationship map `.doqqy/topics.yaml` (Pass 1 regex references + Pass 2 embedding cosine).

```powershell
doqqy map
doqqy map --pass1                # regex only (no store needed)
doqqy map --pass2                # cosine only
doqqy map --threshold 0.80       # stricter thematic links (default 0.75)
doqqy map --top-n 3              # max neighbors per section (default 5)
doqqy map --tag erp12            # thematic pass restricted to a tag (must match ^[\w-]+\Z)
```

### `doqqy index`

Render `.doqqy/topics.yaml` as `processed/INDEX.md` (Obsidian entry point).

```powershell
doqqy index
doqqy index --output ./elsewhere
```

### `doqqy inject`

Inject `[[wikilink]]` blocks (from `topics.yaml`) into the end of each `processed/*.md`, between `<!-- doqqy:links:start/end -->` markers. Idempotent — re-runs replace the previous block.

```powershell
doqqy inject --dry-run           # preview what would change
doqqy inject
```

### `doqqy tags`

List all tags present in the store (i.e. the folder names under `raw/` at embed time).

```powershell
doqqy tags
```

### `doqqy info`

Pipeline state overview: file counts in `raw/` and `processed/`, chunk count, store presence.

```powershell
doqqy info
```

## 4. Typical workflows

### Fresh corpus, end to end

```powershell
mkdir my-corpus; cd my-corpus
mkdir raw
# ... copy documents into raw/ (use subfolders per project/topic) ...
doqqy ingest
doqqy chunk
doqqy embed
doqqy map
doqqy index
doqqy inject
doqqy query "first question"
```

### Documents added or changed

There is no incremental mode yet — rerun the pipeline (ingest is the only slow re-run for big PDF sets):

```powershell
doqqy ingest && doqqy chunk && doqqy embed
doqqy map && doqqy index && doqqy inject      # only if you use the map/Obsidian features
```

### Chunking-config experiment (no re-ingest needed)

```powershell
# edit src/doqqy/config.py → CHUNK_MAX_TOKENS
doqqy chunk
doqqy embed
```

### Obsidian integration

Open **`processed/`** as an Obsidian vault. `INDEX.md` is the entry page; injected `[[wikilinks]]` produce the graph view. Re-run `doqqy map && doqqy index && doqqy inject` after re-embedding to refresh the graph.

## 5. Python API

The CLI is a thin layer — every stage is importable. Construct a `Workspace` pointing at the corpus directory and pass it to each stage; no `chdir`, no import-order constraints, and one process can work with several corpora at once.

```python
from pathlib import Path

from doqqy.workspace import Workspace
from doqqy.ingest import ingest_directory
from doqqy.chunk import chunk_directory
from doqqy.embed import build_index
from doqqy.query import search
from doqqy.map_gen import generate_map
from doqqy.index_gen import generate_index
from doqqy.wikilink_inject import inject_links

ws = Workspace(Path("path/to/my-corpus"))
ws.ensure_dirs()

result = ingest_directory(ws)               # IngestResult(succeeded, failed, skipped)
chunks = chunk_directory(ws)                # list[Chunk], also writes chunks.parquet
n = build_index(ws)                         # int: rows written to LanceDB

hits = search(ws, "JWT refresh flow", k=5, rerank=True, tag="erp12")
for h in hits:                              # list[SearchHit]
    print(f"{h.score:.3f}  {h.source}  {' > '.join(h.section_path)}")
    print(h.content[:200])

generate_map(ws, cosine_threshold=0.8, top_n=3)  # → .doqqy/topics.yaml
generate_index(ws)                          # → processed/INDEX.md
inject_links(ws, dry_run=True)              # InjectionResult(updated, skipped, total_links)
```

Reading the store directly:

```python
import lancedb
from pathlib import Path

from doqqy.config import LANCE_TABLE
from doqqy.workspace import Workspace

ws = Workspace(Path("path/to/my-corpus"))
db = lancedb.connect(ws.store_dir)
tbl = db.open_table(LANCE_TABLE)
df = tbl.search().where("tags_str LIKE '%,erp12,%'").limit(100).to_pandas()
```

## 6. Troubleshooting / FAQ

**`store.lance yok — önce 'doqqy embed' çalıştır`** — you're in the wrong directory, or you haven't embedded yet. Check `doqqy info`.

**Query is slow the first time** — model load (bge-m3 + reranker) happens once per process. Subsequent queries in the same process are fast; each new CLI invocation is a new process. (A serve mode is on the roadmap.)

**A PDF produced garbage / empty output** — probably a scanned (image-only) PDF; doqqy has no OCR. docling errors fall back to pymupdf4llm automatically; if both fail the file lands in the ingest failure list.

**DOCX tables look broken** — mammoth (the fallback) is weaker than pandoc for tables. Check the file's frontmatter `parser:` field; if it says `mammoth`, install pandoc (`winget install pandoc`) and re-ingest.

**Turkish characters garbled on Windows** — the CLI reconfigures stdout to UTF-8 automatically; if piping output, set `$env:PYTHONIOENCODING = "utf-8"`.

**`tags` shows nothing / tag filter returns nothing** — tags are captured at **ingest** time from the folder structure and stored at **embed** time. If you restructured `raw/`, re-run ingest → chunk → embed. Also note tags are exact folder names (case-sensitive) and must match the pattern `^[\w-]+\Z` (Unicode letters including Turkish, digits, underscores, or hyphens). Tag values not matching this pattern will be rejected with an `InvalidTagError` before any store query is issued.

**Out-of-memory during embed** — batch size is already small (4). Lower `EMBEDDING_BATCH_SIZE` in `config.py`, or set `DOQQY_DEVICE=cpu` (GPU VRAM is usually the constraint).

**Where are the logs?** — `.doqqy/logs/` (`ingest.log` for ingest failures).
